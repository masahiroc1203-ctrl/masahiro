# 動画自動編集ツール 設計書・工程表

## 概要

工場工程の連続撮影動画（サイクリック動作）を入力とし、  
各サイクルから指定区間を切り出し、ナンバリング付きで結合して出力するツール。

## サイクル検出アルゴリズム詳細

### 手動モード（推奨・最初の実装）
1. ユーザーが「基準フレーム」を1枚指定（例: ロボットアームがホーム位置にある瞬間）
2. 全フレームと基準フレームの類似度を計算
3. 類似度が閾値を超えた点 = サイクル開始点として検出

### 自動モード（Phase 2以降）
1. 代表的な特徴量（輝度ヒストグラム）の時系列を抽出
2. 自己相関 or FFTで周期を推定
3. 最もスコアが高い繰り返しフレームを自動検出

### 類似度計算方法
- **ヒストグラム比較**: 高速・照明変化に強い
- **特徴点マッチング（ORB）**: 部分一致に強い
- **構造的類似度（SSIM）**: 精度が高いが重い

## 設定パラメータ設計（config.yaml イメージ）

```yaml
input:
  video_path: "./factory_video.mp4"
  reference_frame: 150          # 基準フレーム番号 (手動モード)
  # auto_detect: true           # 自動検出モード

cycle:
  similarity_threshold: 0.92   # 類似度閾値 (0〜1)
  min_cycle_frames: 30          # 最小サイクル長 (フレーム数)

extraction:
  start_offset_sec: 2.0         # サイクル開始から何秒後を切り出し開始
  end_offset_sec: 8.0           # サイクル開始から何秒後を切り出し終了

overlay:
  enabled: true
  position: "top-left"          # top-left / top-right / bottom-left / bottom-right
  font_scale: 1.5
  color: [255, 255, 255]        # RGB
  show_timestamp: true

output:
  video_path: "./output.mp4"
  fps: null                     # null = 入力と同じ
  bitrate: "5M"
```

## 実装上の注意点・リスク

| リスク | 対策 |
|--------|------|
| 照明変化でサイクル検出がずれる | ヒストグラム正規化 + 閾値調整UIの提供 |
| フレームレートが不均一な動画 | FFprobeで事前チェック |
| 非常に長い動画でメモリ不足 | フレームをストリーミング処理（逐次読み込み） |
| 類似フレームが複数ある誤検出 | 最小サイクル長フィルタで除去 |
| 切り出し区間がサイクル境界をまたぐ | バリデーションとエラーメッセージ |

## 最初に実装するMVP（最小構成）

```
CLI引数で動画パスと設定を渡す
  ↓
基準フレーム番号をユーザーが指定
  ↓
ヒストグラム類似度でサイクル開始点を検出
  ↓
各サイクルから指定秒数区間を切り出し
  ↓
ナンバリングテキストを合成
  ↓
FFmpegで結合して output.mp4 を出力
```
